# F010 - 设备端工作台 (Device HUD)

## 1. 功能概述

**设备端工作台 (Device HUD)** 是一个专门为"无头" (Headless) 设备设计的可视化监控页面。它以赛博朋克 (Cyberpunk) 风格的仪表盘形式，实时展示设备的运行状态、底层指令执行情况和硬件性能指标。

### 1.1 目标
- **可视化 Agent 行为**: 让用户直观地看到设备端的 Agent 正在做什么（如下载指令、执行任务）。
- **沉浸式体验**: 提供一种"连接到设备内部"的视觉体验，增强系统的科技感。
- **实时监控**: 提供秒级的 CPU、内存和任务状态监控。

## 2. 核心功能

### 2.1 仪表盘布局 (HUD Layout)
采用深色模式 (Dark Mode) 设计，包含以下视觉元素：
- **Neon 边框与光效**: 青色 (Cyan) 主色调，配合动态扫描线背景。
- **响应式栅格**: 适配不同屏幕尺寸，保证在大屏和小屏上都有良好的展示效果。

### 2.2 系统状态环 (System Status Ring)
- **动态呼吸灯**: 中央的同心圆环根据设备健康度旋转。
- **健康评分**: 根据 CPU 和内存负载计算出的实时健康分数 (0-100)。
- **状态指示**: 清晰显示 `Online` / `Working` / `Idle` 状态。

### 2.3 性能波形图 (Resource Monitor)
- **可视化图表**: 使用 SVG 绘制的实时 CPU 和内存使用率曲线。
- **实时更新**: 每 2 秒同步一次最新数据。

### 2.4 指令日志终端 (Log Terminal)
- **模拟黑客终端**: 滚动的绿色代码流。
- **混合日志源**: 
  - 核心系统日志 (Connection, Ping)
  - 角色特定日志 (根据设备 Role 生成，如 `compiling`, `deploying`)

## 3. 技术实现

### 3.1 架构
- **独立路由**: `/device/:id/hud`，与主应用逻辑解耦，可独立作为监视屏使用。
- **前端模拟**: 为了减少后端存储压力，高频的日志流在前端根据设备状态生成 (Mock)，而核心状态数据 (CPU/Mem/Role) 则从后端 API 实时拉取。

### 3.2 关键组件
- `HUDLayout`: 全局样式容器。
- `StatusRing`: SVG 动画组件。
- `ResourceGraph`: SVG Path 实时绘图组件。
- `LogTerminal`: 自动滚动的日志容器。

## 4. 交互入口
用户可以通过 **设备管理页面** 的设备菜单，点击 **"工作台"** 按钮在新标签页打开该页面。
